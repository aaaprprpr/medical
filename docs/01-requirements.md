# 需求说明

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 项目名称 | 医学影像二分类系统原型 |
| 文档版本 | v0.2 |
| 当前阶段 | 本地原型开发 |
| 适用范围 | 学习 Java Web、前后端分离、模型服务调用和数据库持久化 |

## 2. 项目背景

本项目基于已有 Python 医学影像二分类模型仓库，搭建一个 Java Web 医学系统原型。

系统不用于真实医疗诊断，只用于学习完整工程流程，包括：

- Vue 前端页面开发。
- Spring Boot 后端接口开发。
- Java 后端调用 Python 模型服务。
- 医学影像文件夹上传和解析。
- 患者信息与检测记录数据库练习。

## 3. 当前真实输入格式

前端选择的是一个患者根目录，推荐目录结构如下：

```text
Patient_001/
  Cine/
    SA/
      Location_01/
        Frame_01.png
        Frame_02.png
      Location_02/
        Frame_01.png
  LGE/
    Location_01.png
    Location_02.png
```

兼容规则：

- `Cine`、`LGE`、`SA` 大小写不敏感。
- `SA` 中间层可以不存在。
- Cine 图像按 `Location_xx/Frame_xx.png` 识别。
- LGE 图像按 `Location_xx.png` 识别。
- 前端上传给后端时必须保留相对路径，例如 `Patient_001/Cine/SA/Location_01/Frame_01.png`。

## 4. 项目目标

### 4.1 当前阶段目标

- 前端完成患者文件夹选择、解析和图像浏览。
- 前端将完整患者文件夹中的 Cine/LGE 多文件上传给 Java 后端。
- Java 后端接收多文件，并保留相对路径转发给 Python 模型服务。
- Python 模型服务接收多文件，还原临时患者目录并执行真实模型推理。
- 前端展示预测结果。

### 4.2 后续阶段目标

- Java 后端保存患者信息。
- Java 后端保存检测记录。
- 支持检测历史查询。
- 支持患者信息页面和个人中心练习页面。
- 数据库从简单原型逐步扩展。

## 5. 用户角色

| 角色 | 说明 |
| --- | --- |
| 操作用户 | 本地使用系统录入患者信息、上传影像、查看检测结果的人 |
| Java 后端 | 接收前端请求，负责业务编排、数据保存和模型服务调用 |
| Python 模型服务 | 封装真实模型推理，对 Java 后端提供计算服务 |

## 6. 核心业务流程

当前阶段的检测流程：

```text
用户选择患者文件夹
 -> Vue 前端解析 Cine/LGE 图像
 -> 用户点击开始检测
 -> Vue 前端 multipart 上传多文件到 Java 后端
 -> Java 后端转发多文件到 Python /predict
 -> Python 临时还原患者目录并执行模型推理
 -> Python 返回二分类结果和概率
 -> Java 后端统一包装响应
 -> Vue 前端展示结果
```

数据库接入后的完整流程：

```text
录入患者信息
 -> 保存 patient
 -> 上传患者影像并检测
 -> 保存 test_record
 -> 查询历史检测记录
```

## 7. 功能需求

### 7.1 影像检测工作台

- 支持选择患者文件夹。
- 支持解析 Cine 和 LGE 图像。
- 支持 Cine/LGE 模式切换。
- 支持 Cine Location 列表切换。
- 支持缩略图相册。
- 支持单图查看、上一张、下一张、关闭。
- 支持点击开始检测并展示模型结果。

### 7.2 模型服务调用

- Java 后端通过 HTTP 调用 Python 模型服务。
- 前端不直接调用 Python 模型服务。
- Java 后端不得暴露模型权重路径给前端。
- Python 模型服务负责加载模型和推理。
- Python 模型服务只作为计算服务，不负责数据库保存。

### 7.3 患者和检测记录

后续需要支持：

- 新增患者。
- 查询患者列表。
- 保存检测记录。
- 查询检测历史。

## 8. 非功能需求

- 本地开发阶段允许前端、Java 后端、Python 服务分开启动。
- 接口返回格式尽量保持统一。
- 文件上传大小需要后续通过 Spring Boot 配置控制。
- 第一阶段不追求高并发。
- 第一阶段不处理真实医疗合规问题。

## 9. 当前限制

- 当前还没有正式数据库持久化。
- 当前 Java 后端仍有早期 `Mock` 命名，需要整理。
- 浏览器刷新后不能自动恢复本地文件夹选择，这是浏览器安全限制。
- 模型推理速度不是当前优化重点。
