# 需求说明

## 1. 项目定位

本项目是一个医学影像二分类系统原型，用来练习完整工程流程：

- Vue 前端页面开发。
- Spring Boot 后端接口开发。
- Java 后端调用 Python 模型服务。
- MySQL 数据库增删改查、搜索、排序和后续联表查询。
- 接口文档和开发文档维护。

本项目不用于真实医疗诊断，不处理生产环境部署、权限合规和真实医疗责任问题。

## 2. 核心业务

用户在前端选择一个患者影像文件夹，系统解析 Cine/LGE 图像，调用模型服务进行二分类检测，并展示预测结果。后续系统会保存患者信息和检测记录，用来练习数据库操作和历史记录查询。

## 3. 影像输入格式

前端选择的是一个患者根目录，推荐结构：

```text
Patient_001/
  Cine/
    SA/
      Location_01/
        Frame_01.png
        Frame_02.png
      Location_02/
        Frame_01.png
  LGE/
    Location_01.png
    Location_02.png
```

兼容规则：

- `Cine`、`LGE`、`SA` 大小写不敏感。
- `SA` 中间层可以不存在。
- Cine 图像按 `Location_xx/Frame_xx.png` 识别。
- LGE 图像按 `Location_xx.png` 识别。
- 上传给 Java 后端时必须保留相对路径，例如：

```text
Patient_001/Cine/SA/Location_01/Frame_01.png
Patient_001/LGE/Location_01.png
```

## 4. 已实现功能

### 4.1 影像检测工作台

- 选择患者文件夹。
- 自动提取患者文件夹名。
- 解析 Cine 和 LGE 图像。
- 显示 Cine/LGE 切换。
- 显示 Cine Location 列表。
- 显示缩略图相册。
- 点击缩略图进入单图查看。
- 单图查看支持上一张、下一张和关闭。
- 上传全部解析出的图像到 Java 后端。
- 显示模型判断和置信度。

### 4.2 模型服务调用

- 前端只调用 Java 后端。
- Java 后端调用 Python FastAPI。
- Python 服务只负责模型推理，不访问数据库。
- 模型权重路径不暴露给前端。
- 上传文件在 Python 服务中临时还原成患者目录，推理结束后自动清理临时目录。

### 4.3 患者信息数据库

当前已完成患者表基础能力：

- 新增患者。
- 查询患者列表。
- 根据 ID 查询患者。
- 修改患者。
- 删除患者。
- 按姓名关键词搜索。
- 按性别筛选。
- 按指定字段排序。

## 5. 待实现功能

### 5.1 患者信息页面

目标界面：

- 左侧患者信息表格。
- 右侧搜索和编辑面板。
- 表格展示姓名、性别、年龄、最近检测结果、最近检测时间、最近图像、操作按钮。
- 前端通过 HTTP 接口调用 Java 后端，不在前端完成正式过滤和排序逻辑。

### 5.2 检测记录

后续新增 `test_records` 表，用来保存：

- 患者 ID。
- 检测结果。
- 置信度。
- 检测图像或患者文件夹摘要。
- 检测时间。

患者列表中的“最近检测结果”应通过患者表和检测记录表联表查询得到。

### 5.3 个人中心和登录注册

仅作为后续练习模块，当前不影响核心链路。

## 6. 非功能要求

- 第一阶段只做本地开发，不考虑部署。
- 第一阶段不追求高并发。
- 接口返回格式尽量统一。
- 数据库只保存业务数据和路径摘要，不保存模型权重。
- 图像二进制是否长期保存后续再决定。
