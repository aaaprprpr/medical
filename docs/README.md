# 医学影像二分类系统原型文档

本文档用于记录当前医学影像二分类系统原型的需求、架构、接口、数据库和开发计划。

项目目标不是直接做成生产系统，而是通过一个完整前后端分离项目练习工程化开发流程。

## 文档目录

| 文件 | 说明 |
| --- | --- |
| `01-requirements.md` | 需求说明，描述系统要解决什么问题 |
| `02-architecture.md` | 架构说明，描述前端、Java 后端、Python 模型服务之间的关系 |
| `03-api.md` | API 设计，描述前后端和模型服务接口契约 |
| `04-database.md` | 数据库设计，描述患者和检测记录表 |
| `05-dev-plan.md` | 开发计划，记录当前阶段和下一步工作 |

## 当前阶段

当前项目已经完成了前端检测工作台雏形：

- Vue 前端可以选择患者文件夹。
- 前端可以解析 `Cine/SA/Location_xx/Frame_xx.png` 和 `LGE/Location_xx.png`。
- 前端可以展示 Cine/LGE 图像相册、Location 列表、单图查看器。
- Python 模型服务已经从假接口开始切换到真实模型推理服务。
- Java 后端仍处于早期中转接口阶段，需要从单文件假预测接口整理为正式多文件预测接口。

下一阶段重点：

1. 对齐前端、Java 后端、Python 模型服务的多文件上传链路。
2. 整理 Java 后端包结构和命名，去掉 `Mock` 命名。
3. 再开始接入患者和检测记录数据库。
